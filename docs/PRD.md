# 產品需求文件 (PRD)：食譜收藏夾系統

## 1. 專案概述
- **背景與動機**：日常生活中，人們常在不同平台看到喜歡的食譜，或是打開冰箱看著剩餘食材卻不知道能煮什麼。本系統希望能解決食譜四散不易管理的問題，同時解決「不知道今天要煮什麼」的痛點。
- **目標用戶**：一般人（有烹飪需求、期望記錄或尋找食譜的人）、管理員（維護平台內容與系統管理）。
- **核心價值主張**：提供一個直覺、方便的個人食譜儲存空間，並首創「食材組合搜尋」功能，最大化幫助使用者利用手邊食材。

## 2. 功能需求
以下列出系統的主要功能及對應的使用者故事：

1. **儲存食譜（新增與編輯）**
   - 作為 **一般人**，我希望 **能夠手動新增食譜（包含標題、食材、步驟或圖片）**，以便 **把好用的食譜長久保存下來**。
2. **瀏覽、搜尋食譜**
   - 作為 **一般人**，我希望 **能瀏覽所有食譜，並透過關鍵字搜尋特定料理名稱**，以便 **快速找出想做的菜**。
3. **從食材組合搜尋食譜**
   - 作為 **一般人**，我希望 **能輸入或勾選多種現有的食材來反向搜尋食譜**，以便 **有效消耗冰箱裡的剩餘食材**。
4. **帳號註冊與登入管理**
   - 作為 **一般人**，我希望 **能註冊獨立帳號並登入**，以便 **擁有個人專屬的私人收藏夾空間**。
5. **內容與使用者管理**
   - 作為 **管理員**，我希望 **能查看並管理（包含刪除不當內容）所有公開食譜與使用者**，以便 **維護平台良好的內容品質**。

## 3. 非功能需求
- **技術限制**：以 Flask + Jinja2 為框架，搭配 SQLite 作為關聯式資料庫。
- **效能與安全考量**：
  - 密碼需進行雜湊（Hash）處理，不可明碼儲存。
  - 需要防範常見網頁攻擊（如 SQL Injection, XSS）。
  - 對於查詢回傳的資料筆數需要有良好的分頁或載入機制，確保伺服器不會過載。

## 4. MVP 範圍

### Must Have (必須具備 - 第一階段核心)
- 帳號註冊、登入與登出基本機制。
- 新增、修改、刪除個人食譜。
- 顯示所有公開食譜列表，並提供關鍵字搜尋。
- 提供「輸入多個食材」搜尋對應食譜的核心功能。

### Should Have (應該具備 - 第二階段優化)
- 食物類別/分類標籤（如：中式、便當、甜點）。
- 食譜圖片上傳或圖片網址預覽功能。
- 在「食材組合搜尋」結果中，顯示符合度或缺少的食材。

### Nice to Have (期望具備 - 第三階段亮點)
- 社群互動：可以給別人的食譜按讚或一鍵加入自己的收藏夾。
- 匯出單篇食譜成 PDF 或圖文卡片，便於分享。
